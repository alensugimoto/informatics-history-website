Linux Distributions
Since the Linux kernel is free and open-source (only protected by the GNU General Public License), anyone can download and modify it. Since the beginning developers started to build and release different versions of Linux called distributions or distros for short.

Distributions are software collections which maintain for each software the list of dependencies. A program called package-manager simplifies installation and removal of components.

Distros are different: certain distros are more user friendly and easier to install than others that may required a significant Linux knowledge in order to use it. Other distributions are lightweight and they are more suitable for older computers with slower hardware.

Debian

Debian was one of the first distros, it was released in 1993. It is a free and open-source distribution with more than 50 000 software packages. It can be used as a server or a desktop. Debian is easy to use but a bit difficult to set up.
The package manager is called dpkg.

Debian inspired a lot of other popular distribution, such as Ubuntu and Mint.

Ubuntu

Ubuntu was released in 2004 and it is now the most popular distro. It is very user friendly and it now uses the GNOME desktop environment.

Ubuntu package manager is derived from dpkg and called apt. To simplify installation they developed a graphics installer: Ubuntu Software Center. Now they use Gnome Software from GNU renamed as Ubuntu Software.

Arch Linux

Arch Linux was released in 2002. The creators say that Arch follows the KISS philosophy (Keep it Simple, Stupid): elegance, code correctness, minimalism and simplicity.

It is a rolling distro, so the Arch OS has no versions and all packages are updated on their own. This is a distro recommended for experts. While using it you are forced to learn how things work in Linux. Fortunately everything is explained by a very large community: Arch Wiki.

Red Hat

The Red Hat project started in 1995 to create commercially supported distributions on various platforms. In 2003 they split development in two directions: Red Hat Enterprise Linux (RHEL) for servers and desktops in commercial environments and Fedora for home use.

RHEL is a corporate version, not free. Its releases are slower to be published but with official support and training. CentOS is a community version of RHEL. Quite similar but the support comes from the community.
Fedora is a free community-based distro. It is faster to update but more unstable.

Their original package manager was RPM.
